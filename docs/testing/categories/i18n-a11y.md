# Test category — i18n & Accessibility

**Status:** 📋 P1 (CI lint steps).
**Owner:** `ci-cd-engineer` (lint config); `feature-component-builder` + `route-page-builder` (fix findings).
**Runner:** ESLint plugins + `axe-core` (inside Vitest component tests + Playwright pages).
**Location:** `.github/workflows/lint.yml`, `.eslintrc.cjs`.

## Purpose

Catch i18n drift and a11y regressions at the lint level. Stop bad patterns before they ship; integrate runtime a11y checks into existing test runs.

## i18n checks

### Lint rules (P1)

| Rule | Catches |
|---|---|
| `react-i18next/no-literal-string` (configured for next-intl equivalence) | Hard-coded user-visible strings in JSX |
| Custom rule: `no-missing-message-key` | `useTranslations('foo')('bar')` where `bar` doesn't exist in `messages/{tr,en}/foo.json` |
| Custom rule: `messages-tr-complete` | TR file has every key EN file has (TR is the primary; missing TR = bug) |
| Custom rule: `messages-en-soft` | EN file may have fewer keys but warns when below 80% coverage |

### Configuration

Custom rules in `.eslintrc.cjs` reference `messages/` files. CI runs `pnpm lint`.

### Common offenders to catch

```tsx
// ❌ caught by no-literal-string
<Button>Save</Button>

// ❌ caught by no-missing-message-key (key 'cncel' typo)
const t = useTranslations('common');
<Button>{t('cncel')}</Button>

// ✅
const t = useTranslations('common');
<Button>{t('save')}</Button>
```

## Accessibility checks

### Component-level (every test file with React Testing Library)

```ts
import { axe } from 'vitest-axe';

it('has no a11y violations', async () => {
  const { container } = render(<MyComponent />);
  expect(await axe(container)).toHaveNoViolations();
});
```

Required for every Layer 0 primitive and every Layer 1 pattern (see `docs/testing/categories/frontend-component.md`).

### Page-level (Playwright)

```ts
import { test } from '@playwright/test';
import AxeBuilder from '@axe-core/playwright';

test('marketplace page has no critical a11y violations', async ({ page }) => {
  await page.goto('/pazar-yeri');
  const results = await new AxeBuilder({ page }).analyze();
  expect(results.violations.filter(v => v.impact === 'critical')).toEqual([]);
});
```

Runs in the E2E suite. P1 blocks on `critical` violations; warns on `serious`/`moderate`.

### Manual a11y review (no automation can catch this)

Quarterly, `tester` walks the critical paths with:

- Keyboard only (Tab order, focus rings, escape from modals).
- Screen reader (VoiceOver on macOS, NVDA on Windows).
- Browser zoom at 200%.
- High-contrast mode.

Findings logged in `docs/testing/a11y-manual-reviews/<date>.md`.

## Non-negotiables

- ❌ Never hard-code Turkish or English strings outside `messages/`.
- ❌ Never use `aria-hidden="true"` to hide problematic content from a11y tests.
- ❌ Never ignore an axe `critical` finding without a documented mitigation.
- ✅ Always run a11y in component tests.
- ✅ Always set `lang="tr"` on the html element (default) with `lang="en"` switching when the user changes locale.
