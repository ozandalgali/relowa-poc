# Test category — Contract

**Status:** 📋 P1 light (CI check) · full P2.
**Owner:** `tester`.
**Runner:** Vitest + `@hono/zod-openapi`.
**Location:** `tests/contract/`.

## Purpose

Verify that the OpenAPI spec (auto-generated by `@hono/zod-openapi`) matches the actual route behavior, and that breaking changes are surfaced before they ship.

## What this covers

- Every Hono route has a Zod schema for input + output.
- The generated OpenAPI document is valid (parseable, no missing operationIds).
- Breaking changes (removed fields, changed types) are caught by diff against a committed snapshot.

## Test shape (P1)

```ts
import { describe, it, expect } from 'vitest';
import { app } from '@relowa/api';
import openapiSnapshot from './openapi.snapshot.json';

describe('OpenAPI contract', () => {
  it('every route has an OpenAPI definition', () => {
    const spec = app.getOpenAPIDocument();
    const routes = listAppRoutes(app);
    for (const route of routes) {
      expect(spec.paths[route.path]?.[route.method]).toBeDefined();
    }
  });

  it('matches the committed snapshot (no breaking changes)', () => {
    const spec = app.getOpenAPIDocument();
    expect(spec).toMatchSnapshot();
  });

  it('Idempotency-Key parameter is documented on every mutation', () => {
    const spec = app.getOpenAPIDocument();
    const mutations = collectMutations(spec);
    for (const op of mutations) {
      const params = op.parameters ?? [];
      expect(params.some(p => p.name === 'Idempotency-Key')).toBe(true);
    }
  });
});
```

## Snapshot drift

When the spec changes intentionally:

```bash
pnpm test -- -u tests/contract/openapi.test.ts
```

The PR review then audits the snapshot diff. Unintentional breaking changes get caught here before they reach a consumer (frontend, partner integration).

## P2 expansion

- **Consumer-driven contract tests.** Frontend records expectations; backend verifies them.
- **Provider mocks** generated from the OpenAPI spec so frontend tests run without a backend.
- **Versioned contracts** when v2 APIs land.

## Non-negotiables

- ❌ Never commit a snapshot diff without explaining the intent in the PR description.
- ❌ Never remove an OpenAPI definition from a route without versioning the API.
- ✅ Always include `Idempotency-Key` in mutation specs.
